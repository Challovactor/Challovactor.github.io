一台 GPU 服务器从“能用”到“好用”，中间隔着非常多工程细节。

<!--more-->

## 1. 先定资源规则

至少要明确：

- 谁可以长期占卡
- 谁可以跑夜间任务
- 哪些目录是公共目录
- 哪些目录必须私有

没有规则时，团队最容易陷入相互覆盖和相互影响。

## 2. 环境必须隔离

共用机器时，我强烈建议每个项目独立环境，至少做到：

- 独立 Python 环境
- 独立日志目录
- 独立输出目录

否则一个人升级依赖，很容易把别人跑好的实验打坏。

## 3. 数据与模型要分层存放

推荐把存储拆成：

- 公共原始数据
- 项目中间数据
- 模型权重仓
- 实验结果目录

这样做的好处是权限更清晰，清理空间时也不容易误删。

## 4. 日志和监控不要缺

共享机器至少应该知道：

1. 谁在跑任务
2. 哪张卡被谁占用
3. 哪个进程最耗显存
4. 磁盘什么时候快满了

## 5. 机器治理也是工程能力

真正成熟的团队，不会把服务器当作“谁先连上谁先用”的公共电脑，而会把它当作可维护的研发基础设施。
