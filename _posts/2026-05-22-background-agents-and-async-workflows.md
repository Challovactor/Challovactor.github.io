大家最熟悉的 Agent 形态是聊天框，但很多真实业务任务并不适合在一次对话里完成。

<!--more-->

## 1. 哪些任务更适合异步

例如：

- 长时间资料搜集
- 多步骤处理
- 批量生成
- 定时巡检

## 2. 异步系统更需要状态管理

至少要有：

1. 任务状态
2. 进度更新
3. 失败重试
4. 结果回传

## 3. 用户感知要重新设计

后台任务不是没有交互，而是要换一种交付方式。

## 4. 这类系统比聊天 Agent 更像生产系统

它会更关注可靠性、幂等性和可追踪性。

## 5. 后台 Agent 的价值在于持续工作

一旦系统能脱离实时对话独立运行，它的产品价值会明显提升。
