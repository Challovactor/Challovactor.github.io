很多人写博客断更，不是因为没东西写，而是因为选题、草稿和发布之间没有一个稳定的工作流。

<!--more-->

## 1. Issue 最适合承接零散想法

我通常会把下面这些都放进 Issue：

- 选题标题
- 想写的核心问题
- 准备引用的案例
- 读者可能会遇到的坑

## 2. 给选题做轻量状态管理

最简单的状态就够用了：

1. `idea`
2. `drafting`
3. `published`

这样不会太重，但能看见内容池是否在增长。

## 3. 一篇文章最好回答一个明确问题

比如：

- 环境为什么总是冲突
- Agent 为什么一上线就不稳定
- 检测项目为什么指标好看却不好用

问题越清晰，文章越容易写得有价值。

## 4. 发布后不要关闭反馈链路

Issue 的另一个好处是，你可以继续收集：

- 评论里的新问题
- 后续更新点
- 配套文章计划

## 5. 博客系统真正重要的是“可持续”

如果一个工具能让你持续把经验转成文章，它就不仅是记录工具，也是长期复利工具。
