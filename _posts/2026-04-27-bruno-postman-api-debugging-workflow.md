接口联调最怕的不是接口本身复杂，而是团队没有一套统一的调试资产。

<!--more-->

## 1. 至少管理三类信息

我建议工具里固定维护：

- 请求样例
- 环境变量
- 错误场景样例

## 2. 不要只存成功请求

很多线上问题都出在失败路径上，例如：

- 缺参数
- 权限不足
- 超时
- 非法文件格式

## 3. 环境切换一定要干净

开发、测试、生产前的预发环境最好分开，否则特别容易误调用。

## 4. AI 接口更要重视结构

尤其是大模型服务，最好先在调试工具里验证：

1. 输入格式
2. 输出 schema
3. 错误返回
4. 超时策略

## 5. 一套好的联调资产会复用很久

它会服务于开发、测试、排障和后续新人接入，而不只是一次性调试。
