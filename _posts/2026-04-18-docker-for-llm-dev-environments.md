很多人一提到 Docker，就先想着怎么把镜像做小。但在大模型项目里，更关键的问题其实是“边界”。

<!--more-->

## 1. 先确定容器里放什么

我更建议把环境拆成三层：

- 基础镜像：系统、Python、CUDA
- 应用镜像：依赖与服务代码
- 运行配置：模型地址、密钥、存储路径

不要把所有东西都烤进一个镜像里。

## 2. 模型权重尽量外挂

权重通常体积大、变更慢、复用高。把它们直接打进镜像，会导致：

- 构建慢
- 推送慢
- 回滚重

更实用的方式是挂载目录或从对象存储拉取。

## 3. 本地开发和线上运行要分开

开发镜像可以更宽松，允许：

- 调试工具
- Notebook
- 额外系统包

线上镜像则更关注：

- 启动速度
- 依赖可控
- 安全与可回滚

## 4. 不要忽略 GPU 与权限问题

很多 Docker 方案在 CPU 上能跑，但一到 GPU、挂载卷和网络权限就出问题。

上线前至少确认：

1. GPU 可见
2. 模型目录有读权限
3. 输出目录有写权限
4. 端口与健康检查正常

## 5. 好的容器化结果是什么

不是“镜像能启动”，而是“项目成员、测试环境和部署环境对同一套服务边界有一致理解”。
