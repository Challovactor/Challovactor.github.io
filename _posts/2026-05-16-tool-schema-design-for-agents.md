很多 Agent 工程问题，最后都能追到工具 schema 设计不清。

<!--more-->

## 1. 字段命名必须清晰

工具输入不要使用模糊字段名，例如：

- `data`
- `info`
- `value`

最好让字段名直接表达业务含义。

## 2. 输入约束越明确越好

至少要说明：

1. 必填项
2. 可选项
3. 枚举范围
4. 默认值

## 3. 错误返回必须可消费

如果工具失败只返回一段含糊文本，Agent 很难做可靠重试。

## 4. 工具不要承担太多职责

一个工具只做一类动作，会比“大一统万能工具”更容易维护。

## 5. 好 schema 本身就是产品设计

它决定了智能体与系统交互时能否真正稳定。
