目标检测是很多人进入计算机视觉的入口，但真正做项目时，难点往往不在模型结构，而在整个链路。

<!--more-->

如果让我重新做一个检测项目，我会从下面这张清单开始，而不是先挑模型。

## 1. 先定义任务，不要先选网络

先问自己四个问题：

1. 要检测什么目标
2. 在什么场景下检测
3. 结果给谁用
4. 错误成本是什么

例如，无人机巡检里的检测任务和普通互联网图片检测是完全不同的：

- 视角更复杂
- 光照变化更大
- 遮挡更多
- 误报和漏报成本更高

如果这些前提没想清楚，后面调参会非常痛苦。

## 2. 数据集比模型更重要

一个实战项目里，我最看重的是：

- 类别定义是否清晰
- 标注标准是否统一
- 训练集和测试集分布是否一致
- 有没有足够多的难例

很多检测项目精度上不去，不是模型太弱，而是数据集的问题没有被正视。

我建议先抽样看一遍数据：

- 框是不是标得一致
- 小目标比例高不高
- 背景干扰多不多
- 是否存在严重类别不平衡

## 3. 训练前先做一个 baseline

不要一开始就堆复杂结构。先做一个最小可跑 baseline：

1. 选一个成熟框架
2. 跑通数据加载
3. 训练 5 到 10 个 epoch
4. 看 loss、可视化结果和错误案例

这个 baseline 的价值不在于最优，而在于帮你快速判断问题出在哪：

- 数据
- 配置
- 训练过程
- 推理后处理

## 4. 检测项目最值得关注的几个指标

我通常不会只看 mAP。

更实用的指标包括：

- 小目标召回率
- 漏检率
- 高置信度误报数量
- 推理速度
- 在真实场景下的稳定性

尤其是工程场景里，mAP 高不代表系统可用。

## 5. 常见优化路径

当 baseline 跑通后，我通常按下面顺序优化：

1. 先修数据
2. 再调输入尺寸和增强策略
3. 再看 backbone / neck / head
4. 最后再考虑更复杂的 trick

很多人会反过来做，结果花很多时间调模型，却没有改善真正的问题。

## 6. 部署时要关注什么

检测模型从训练走向部署时，经常会遇到这些坑：

- 训练图片尺寸和线上不一致
- 预处理流程前后不一致
- NMS 参数没有同步
- 推理框架转换后结果漂移

我建议把部署前后的同一张图做对比：

1. 原始输入
2. 预处理后输入
3. 模型输出
4. 后处理后的最终结果

只要这四步不能稳定对齐，系统上线后很容易出现“训练效果很好、线上效果很差”的情况。

## 7. 一个更实用的项目观

目标检测项目不是“训一个模型”，而是做一条完整的识别链路：

```text
任务定义 -> 数据采集 -> 标注规范 -> 训练 -> 评估 -> 推理 -> 部署 -> 反馈闭环
```

当你真正把它看成一条链路时，很多问题就会更容易定位。

这也是我为什么始终觉得，计算机视觉最重要的不只是模型，而是工程视角。
